1. Resumen Ejecutivo
Este documento define el plan completo de organización, roles, fases de desarrollo y criterios de inicio para llevar el proyecto desde cero hasta la entrega final en 6 meses. El objetivo principal es tener el 80% del sistema funcional para noviembre, con clientes reales (barberías) validando el producto.

Ítem	Detalle
Equipo	Aguirre, Cabanilla, Filippin — 3 devs avanzados
Duración	24 semanas (6 meses)
Fases	4 fases: Cimientos → Core → Transaccional → Piloto
Stack	NestJS · PostgreSQL · MongoDB · Next.js · React Native
Meta	80% funcional en producción con barberías reales para noviembre

2. División de Roles
Cada integrante es "dueño" de su área pero colabora en las demás. Los roles no son rígidos — se apoyan constantemente, especialmente en la Fase 2.

Dev	Rol	Tecnologías	Responsabilidades clave

Dev 2	Mobile Lead	React Native · Expo	App cliente final: reservas, pagos, gamificación, notificaciones

6. Riesgos Principales y Cómo Mitigarlos

#	Riesgo	Impacto	Mitigación
1	El algoritmo de triangulación de agendas resulta más complejo de lo esperado	Alto — bloquea el flujo principal de reservas	Atacarlo en Fase 2 (semanas 5-7), no al final. Dev 1 y Dev 3 trabajarlo juntos.
2	Mercado Pago tarda en aprobar la cuenta de producción	Medio — retrasa la Fase 4 con dinero real	Iniciar el proceso de alta en MP desde la semana 9, en paralelo con el sandbox.
3	Los clientes reales (barberías) piden features no planificados	Medio — consume tiempo de corrección de bugs	Dejar buffer de 3 semanas en la Fase 4. Decir no a features fuera de alcance.
4	Sin contrato de API acordado, front y back se bloquean mutuamente	Alto — paraliza el desarrollo en paralelo	Definir todos los endpoints en Swagger en la semana 2, antes de codear cualquier pantalla.

7. Reglas del Equipo
Acuerdos mínimos para no perder tiempo en fricciones evitables:

•	Ningún código va directo a main. Todo pasa por una rama feature/xxx → PR → review de al menos 1 compañero → merge a develop.
•	Reunión semanal de 30 minutos todos los lunes: qué hice, qué voy a hacer, qué me bloquea.
•	Si algo tarda más de 2 días sin avance, avisar al equipo antes de seguir solo.
•	Las decisiones técnicas importantes se documentan en Notion el mismo día que se toman.
•	El esquema de base de datos no se modifica sin que los 3 estén de acuerdo.
•	Los commits siguen la convención: feat: / fix: / docs: / refactor: para mantener el historial limpio.
